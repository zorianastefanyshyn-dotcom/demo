Бізнес-логіка вибору відділення

  • Пріоритет джерел зберігається у зовнішній конфігурації, не в Cargos
  • Порядок: NPU → Cargos → Temu → my.nova.global → Legacy
  • Оркестратор обирає перший активний запис за пріоритетом
  • Cargos відображає всі записи нейтрально — без ранжування

Аналітика якості NPU

  • При вимкненні зберігається: причина + коментар + timestamp + хто вимкнув
  • Дозволяє аналізувати частоту скарг по конкретному підряднику
  • Можна порівнювати дату received_at і disabled_at — наскільки швидко дані застаріли
  • Дані не видаляються — лише is_active = false

Редагування по джерелах

  • NPU, Temu, my.nova.global, Legacy — тільки вимкнути/увімкнути
  • Cargos — створити, редагувати, видалити вручну
  • При збереженні Cargos — оновлюється received_at

Мультикраїнність

  • Кожна країна — окрема вкладка
  • Один клієнт може мати різних підрядників у різних країнах (NP в UA, InPost у PL)
  • country_code + carrier_id ізолюють записи між собою
